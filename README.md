# GGAI Stella Engine

> **당신의 옵시디언 볼트가 곧 RP 무대가 된다.** — SillyTavern의 호환성과 NovelAI 스토리텔러의 몰입감을 한 플러그인에 담은, 브랜치형 인터랙티브 스토리텔링 엔진.

Stella Engine은 실리태번·NovelAI에서 "정말 좋았던 것들"만 골라 옵시디언 위에 다시 지은 RP 엔진입니다. 캐릭터 카드, 로어북, 프롬프트 프리셋을 **그대로 가져와** 쓰고, 이야기를 자유롭게 분기시키며, 번역·삽화·요약까지 한 창 안에서 돌립니다. AI는 [GGAI Core](../obsidian-ggai-core)를 통해 원하는 프로바이더(Claude · GPT · Gemini · NovelAI · 로컬 모델)로 골라 붙입니다.

---

## 왜 Stella인가

**1. 당신의 자료를 버릴 필요가 없다.**
실리태번에서 쓰던 캐릭터 카드(JSON·PNG·CHARX), NovelAI 로어북(`.lorebook`)과 시나리오(`.scenario`), 심지어 진행 중이던 NovelAI 스토리(`.story`)까지 그대로 임포트됩니다. 프롬프트 프리셋은 실리태번과 **양방향 호환** — Stella에서 만든 세트를 실리태번으로 다시 내보낼 수도 있습니다.

**2. 이야기가 한 줄이 아니다.**
모든 생성은 분기 트리의 한 노드입니다. 마음에 안 드는 전개는 재생성해서 형제 가지로 남기고, 좋았던 순간은 즐겨찾기(세이브 포인트)로 찍어 두었다가 언제든 되돌아가 다른 길로 이어씁니다. NovelAI 스토리텔러처럼 자연스럽게, 그러나 트리 전체가 보존됩니다.

**3. 클라우드가 아니라 당신의 볼트.**
모든 세션·캐릭터·로어북·삽화가 볼트 안 평범한 파일로 저장됩니다. 외부 서비스에 갇히지 않고, 옵시디언에서 직접 열어 보고, 백업하고, 동기화합니다.

**4. 원하는 AI를 원하는 대로.**
Claude로 서사를 쓰고, NovelAI로 삽화를 뽑고, 로컬 모델로 번역을 돌리는 식으로 **작업마다 다른 모델**을 붙일 수 있습니다. API 키 관리와 프로바이더 연결은 전부 GGAI Core가 담당합니다.

## 핵심 기능

### 🌿 브랜치 세션 — 이야기를 나무처럼 키운다

- **이어쓰기 / 재생성 / 형제 이동**: 재생성해도 이전 결과가 사라지지 않고 형제 가지로 남습니다. 어느 갈래든 자유롭게 오갑니다.
- **patch 기반 저장**: 본문을 통째로 복사하지 않고 편집 델타만 쌓습니다. 10만 토큰까지 길어져도 파일은 가볍게 유지됩니다.
- **직접 편집이 기본**: 세션 본문을 그냥 고치면 즉시 새 노드/variant로 저장됩니다. 읽기 모드·편집 버튼을 오가는 번거로움이 없습니다.
- **세이브 포인트**: 노드 즐겨찾기로 결정적 장면을 표시하고, undo/redo로 안전하게 실험합니다.
- **이어쓰기 이음새 보정**: 챗 모델이 이어쓸 때 마지막 문장을 매끄럽게 물고 시작하도록 앵커를 붙이고, 응답 앞의 반복 문장은 스트리밍 중 자동으로 걷어냅니다.
- **읽던 위치 기억**: 화면 중앙의 노드를 기준으로 스크롤 위치를 복원하고, 생성 중에는 꼬리를 따라가다 위로 스크롤하면 멈춥니다.

### 📥 강력한 임포트 — 커뮤니티 자산 그대로

| 포맷 | 지원 |
|---|---|
| **SillyTavern 캐릭터 카드** | JSON / PNG·APNG (내장 청크) / CHARX (ZIP) |
| **Character Card V3 / V2 / V1** | 자동 판별 → 정규화 |
| **NovelAI 로어북** | `.lorebook` |
| **NovelAI 시나리오** | `.scenario` (본문·내장 로어북·메모리 분리 복원) |
| **NovelAI 스토리** | `.story` (진행분까지 — 문단별 저자(유저/AI) 출처를 복원해 브랜치 노드로 재구성) |

임포트한 것은 전부 Stella의 통합 스키마로 정규화되어, 어디서 왔든 똑같이 동작합니다.

### 📖 로어북 — 실리태번 월드인포 완전 호환

실리태번 월드인포를 기준으로 통일된 로어북 엔진입니다. 커뮤니티가 쓰던 규칙이 그대로 통합니다.

- 키워드 매칭(정규식·대소문자·완전 단어), secondary key + selective logic
- `constant`(항상 주입), `probability`(확률 주입), recursion 제어(exclude/prevent/delayUntil)
- 삽입 위치(`before_char` / `after_char` / `before_examples` / `after_examples` / `at_depth`)와 depth·role·order를 실리태번과 동일한 의미로 해석
- 전용 에디터로 옵시디언 안에서 직접 편집

### 🎭 페르소나 (유저 프로필)

- 여러 페르소나를 만들고 활성 페르소나를 골라 플레이
- **세션이 페르소나를 기억**: 새 세션을 시작한 시점의 페르소나를 세션이 기록해 두었다가, 다시 열면 그 페르소나로 복원합니다.
- **전용 시나리오** 지정: 특정 페르소나를 특정 시나리오에 자동으로 붙일 수 있습니다.
- `{{persona}}` 매크로로 위치를 지정하면 그 자리를, 없으면 캐릭터 설명 앞 기본 위치에 삽입 — 실리태번과 같은 규칙.

### 🧩 프롬프트 프리셋 — 실리태번 양방향 호환

- 기본 세트 2종 제공: **Default** / **Default (NovelAI)** (구성·순서는 동일, Main Prompt 내용만 다름)
- **"NAI 형식으로 보내기"** 토글 하나가 NovelAI식 가공(역할 토큰·본문 구분자 등) 전체를 담당 — 텍스트 모델이면 자동으로 켜지고 챗 모델이면 꺼집니다.
- 마커·매크로·프롬프트 순서를 실리태번 개념 그대로 사용
- **실제 다운로드 익스포트**: 프리셋을 OS 다운로드 폴더로 내보내 실리태번 등 외부 앱에 바로 옮겨 쓸 수 있습니다.

### 🌍 번역 — 문단 단위 번역 메모리

- 문단 + 원문 해시 기준의 **번역 메모리**: 같은 문단은 다시 번역하지 않고, 바뀐 문단만 갱신
- 출력 방식 선택: **원문 치환** 또는 **좌우 2분할**(드래그 분할바 + 스크롤 체인)
- **드래그 선택 영역만 번역** / 재번역, 일괄 번역, 자동 번역(생성 완료 후)
- 번역 보기에서 **직접 편집** 가능 — 고친 문단은 user-edit variant로 저장
- 실행 단위 되돌리기(undo)로 번역 결과를 한 단계씩 복구

### 🎨 삽화 — 장면에 그림을 입힌다

- 본문 맥락에서 프롬프트를 자동 생성 → NovelAI 이미지로 렌더
- 출력 위치: **전용 출력 뷰**(독립 패널 — 팝아웃해서 듀얼모니터 전체화면 가능) 또는 **본문 인라인**
- **variant 캐러셀**로 같은 장면의 여러 그림을 좌우로 넘겨 고르고, 라이트박스로 전체화면 감상
- **갤러리**로 세션 전체 삽화를 한눈에 — 클릭하면 그 장면의 분기로 이동
- 수동(툴바 버튼) / 자동(생성 후, 밀도 게이트) 생성 모두 지원

### 🧠 요약 — 긴 세션의 기억을 유지

- **노드 앵커 누적 요약**: 경로를 따라가며 "사건 기록(불변 누적)"과 "현재 상황 스냅샷(갱신)"을 쌓습니다.
- 분기를 갈아타면 그 경로의 요약이 자동으로 유효해지고, 재생성으로 빠진 구간은 저절로 제외됩니다.
- 밀린 구간은 주기 단위로 쪼개 **조각마다 즉시 저장** — 도중에 창을 닫아도 완료분은 안 날아갑니다.
- **요약 관리 창**에서 사건 조각을 직접 읽고 고치고, 구간별 재생성·전체 재작성이 가능합니다.
- 요약은 작가노트 위 기본 위치에 자동 주입되거나 `{{summary}}` 매크로로 위치 지정.

### 🔁 문단 재생성 — 한 문단만 다시 쓴다

- 문단 선택 모드에서 원하는 문단(범위)을 골라 재생성
- 저장 프롬프트 선택 + 직접 지시(instruction) 조합
- **AI 보조 문단 편집기**: 원문에서 시작해 재생성 결과가 그 자리에 들어오고, 손으로도 다듬어 [적용] — 원문 불변 원칙으로 항상 새 variant 파생

### 🗂️ 대시보드 — 모든 것을 한 허브에서

홈 · 시나리오 · 세션 · 갤러리 · 페르소나 · 로어북 · 프롬프트 **7개 탭**.

- **홈**: "이어서 하기" 히어로 카드(세션 최신 삽화·페르소나 표지 배경) + 추천 시나리오
- **세션 탭**: 모든 시나리오의 세션을 최근 플레이순으로, 이어하기·갤러리·분기 관리·**읽기 모드로 내보내기**(본문+삽화 인라인 `.md`)
- **갤러리 탭**: 전 세션 삽화 집계, 시나리오 필터, 셀에서 바로 그 분기로 이동
- **시나리오 탭**: 태그 칩 필터·검색·정렬, 포스터 그리드, 시나리오 복사
- 검색·태그 탐색·인라인 분기 관리(전체 맵 + 현재 노드 중앙) 내장

### 🧷 확장 시스템 — 엔진을 열어 둔다

Stella는 내장 기능(요약)도 외부 플러그인도 **같은 API**로 동작하도록 설계되었습니다.

- **설정 패널 레지스트리**: 우측 `확장` 탭에 자신의 설정 패널을 꽂을 수 있습니다(`SettingsPanel`).
- **실행 레지스트리**: 컨텍스트 기여, 생성 완료 훅, 로어북 선택 대체 등 생성 과정의 이음새에 개입하는 확장(`StellaExtension`)을 등록할 수 있습니다.
- 요약 기능 자체가 이 확장 위에서 동작하는 참조 구현입니다 — 외부 확장도 같은 방식으로 엔진에 결합됩니다.

## 설치

> **GGAI Core가 먼저 설치·활성화되어 있어야 합니다.**
https://github.com/0ogg/GGAI-CORE

### BRAT (권장)

1. 옵시디언 → 설정 → 커뮤니티 플러그인 → BRAT 설치·활성화
2. BRAT → "Add Beta Plugin" → 이 저장소 주소 입력
3. **GGAI Core**와 **GGAI Stella Engine**을 모두 활성화

### 수동 설치

1. Releases에서 `main.js`, `manifest.json`, `styles.css` 다운로드
2. 볼트의 `.obsidian/plugins/ggai-stella-engine/` 폴더에 복사
3. GGAI Core 활성화 후 Stella Engine 활성화

### 처음 시작하기

1. GGAI Core 설정에서 AI 프로필을 하나 이상 등록합니다(예: Anthropic 또는 NovelAI 키).
2. 리본의 Stella 아이콘으로 대시보드를 엽니다.
3. 캐릭터 카드를 임포트하거나 새 시나리오를 만들고, 세션을 시작해 이어쓰기를 눌러 봅니다.

첫 실행 시 볼트 루트에 `GGAI/` 폴더 구조(`SCENARIOS/`, `LOREBOOKS/`, `PROMPTS/`, `PRESETS/`, `USERS/`)가 자동으로 생성됩니다.

## 의존성

- **GGAI Core** (`>=0.2.0`) — 필수. 모든 AI 호출·프로필·토큰 카운트·스트리밍·에이전트 기능을 제공합니다. Stella는 API 키나 프로바이더 어댑터를 직접 구현하지 않고 전부 Core를 경유합니다.

## 데이터가 저장되는 곳

```
GGAI/
├── SCENARIOS/          # 시나리오 폴더 (scenario.json = CCv3 호환 + SESSIONS/)
│   └── <시나리오>/SESSIONS/<세션>/
│       ├── session.json          # 브랜치 트리 (patch 기반)
│       ├── translations.json     # 문단 번역 메모리
│       ├── illustrations.json    # 노드별 삽화 variant
│       ├── summaries.json        # 노드 앵커 누적 요약
│       └── assets/               # 삽화 PNG 등
├── LOREBOOKS/          # 로어북 (실리태번 월드인포 호환 스키마)
├── PROMPTS/            # 프롬프트 세트 (실리태번 호환)
├── PRESETS/            # 프리셋(북마크)
└── USERS/              # 페르소나 프로필
```

전부 사람이 읽을 수 있는 JSON입니다. 옵시디언에서 직접 열어 확인하거나, 다른 도구로 가공하거나, 그대로 백업할 수 있습니다.
커뮤니티 플러그인: Obsidian Custom File Extensions Plugin을 설치하면 옵시디언 편집기에서도 json에 접근 가능합니다.
